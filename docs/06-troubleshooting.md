# Troubleshooting

This guide covers the most common problems a beginner is likely to hit when running the mobile app locally.

## 1. The app stays on the loading screen

### What usually causes it

- The Firebase values in `.env` are missing or wrong.
- The signed-in user does not have a Firestore profile.
- The profile fetch is timing out.
- The device has no network connection.

### What to check

1. Confirm `.env` has real Firebase values.
2. Confirm the user exists in Firebase Auth.
3. Confirm there is a `users/{uid}` document in Firestore.
4. Check the Metro console for auth or profile errors.

### Helpful detail

`AuthContext` has an 8-second initialization timeout, so the app should eventually stop waiting even if the profile lookup fails.

## 2. Login fails

### Common causes

- Wrong email or password.
- The account was created in the wrong Firebase project.
- The user profile is marked inactive.

### Fixes

- Re-enter the email and password carefully.
- Make sure the account is in the same Firebase project as the app config.
- Check the `status` field in `users/{uid}`. It must be `active`.

## 3. I signed in, but I landed on the wrong dashboard

### Common causes

- The role field in Firestore is wrong.
- The role is missing and the fallback driver role is being used.
- The auth profile is stale.

### Fixes

- Check `users/{uid}.role`.
- Make sure the role is one of `admin`, `owner`, `operator`, or `driver`.
- Sign out and sign back in after fixing the profile.

## 4. The QR scanner does not work

### Common causes

- Camera permission was denied.
- The emulator or device does not expose a usable camera.
- The app does not have access to the hardware camera on the machine.

### Fixes

- Grant camera permission in the system settings.
- On Android emulator, make sure camera support is enabled.
- Use a real device if the emulator camera is unreliable.

## 5. The QR code opens the wrong screen or no screen at all

### Common causes

- The QR payload is not a valid token or URL.
- The deep link is missing the `token` query parameter.
- The link is being opened in a context that does not support the custom scheme.

### Fixes

- Use the QR generated by the operator dashboard.
- If you are testing manually, paste just the token into the driver home screen.
- Use an installed build or dev build when testing `smartparking://` links.

## 6. Google Maps does not open

### Common causes

- `GOOGLE_MAPS_API_KEY` is missing.
- The parking does not have coordinates.

### Fixes

- Add the maps API key to `.env`.
- Make sure the parking record has latitude and longitude values.

The app still shows the selected parking text even when the map preview cannot load.

## 7. Functions calls fail locally

### Common causes

- The web app emulators are not running.
- `USE_FUNCTIONS_EMULATOR` is not set correctly.
- `FUNCTIONS_EMULATOR_HOST` or `FUNCTIONS_EMULATOR_PORT` is wrong.
- The emulator is running, but the host is not reachable from the device.

### Fixes

1. Start the emulators from the `Web App` folder.
2. Confirm the port is `5001` unless you changed it.
3. On Android emulator, `localhost` should resolve correctly, but if it does not, restart the emulator and Metro.
4. Watch the console output from `src/config/firebase.js` for the emulator host message.

## 8. The driver sees no parkings or no bookings

### Common causes

- There are no active parkings in Firestore.
- The signed-in user has no bookings or sessions yet.
- The active data belongs to a different Firebase project.

### Fixes

- Seed or create parking records.
- Confirm the driver profile and parking data are in the same project.
- Use the admin or operator screens to create the data first.

## 9. The operator sees no assigned parkings

### Common causes

- The operator profile does not include `assignedParkingIds`.
- The assigned parkings were deleted or renamed.

### Fixes

- Update the operator profile in Firestore.
- Use the owner or admin screens to reassign parking access.
